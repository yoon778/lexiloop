# LexiLoop 확정 결정 기록

## 제품

- 프로젝트명과 앱 이름은 `LexiLoop`
- Android 모바일 앱
- 1개월 개인 베타 후 수정·보완하여 Play Store 출시 검토
- 영단어 학습에만 집중
- 깔끔함을 UI 모토로 사용

## 개인화

- 자연어로 학습 목적 입력
- AI가 분야, 비율, 난이도 분석
- 최종 확인 팝업 후 생성
- 자가 수준 선택과 20단어 진단 병행
- AI 추천과 검수된 사전 데이터를 혼합
- 초기 300개, 잔여 100개 미만이면 300개 보충 제안

## 학습

- 신규 단어와 복습량 분리
- 적응형 단계 사용
- `안다`는 객관식을 생략하되 철자와 문장으로 검증
- `안다` 검증을 모두 통과하면 첫 복습은 7일 후
- `헷갈린다`와 `처음 본다`는 같은 일반 학습 경로 사용
- 영어 철자는 완전 일치
- 대소문자와 공백 무시
- 한국어 뜻 타이핑 없음
- 문장 빈칸은 영어 타이핑
- 미국식 발음 듣기만 지원
- 따라 말하기와 발음 분석 제외
- 여러 뜻은 목표 뜻 강조, 보조 뜻 흐리게 표시
- 단어, 숙어, 구동사, 개발 용어 포함
- 기본형 중심, 변화형과 파생어 구분

## 복습

- 고정 간격: 당일, 1일, 3일, 7일, 14일, 30일
- 복습은 철자 또는 문장부터 시작
- 예정 복습 21개 이상이면 신규 학습 잠금
- 이미 시작한 신규 세션은 진행 중 복습 적체가 21개가 되어도 유지
- 복습 진입 문제는 간격 인덱스 홀수에서 철자, 짝수에서 문장 빈칸 사용
- 신규 목표 중 이미 아는 단어가 있어도 당일 보충하지 않음
- `나중에 다시`도 카드를 확인한 신규 항목 수에 포함
- `이미 완전히 앎`은 30일 복습 단계로 이동
- 날짜가 바뀐 미완료 세션은 만료하고 미완료 신규 항목은 대기열로 복귀

## 데이터와 운영

- AI 생성만 온라인, 학습은 오프라인
- 로그인 없는 개인 베타
- Room, DataStore, JSON 백업
- Room은 6개 테이블의 최소 모델 사용
- 제외 여부는 학습 수명주기와 독립적으로 저장하여 복원 시 진도 유지
- 목표 뜻과 예문은 `LearningItem`에 포함
- 추천 대기열·복습 일정·제외 목록은 `LearningProgress`에 통합
- ID는 백업 안정성을 위해 UUID 문자열 사용
- 원시 문제 시도 이력은 저장하지 않고 현재 세션 복구값만 저장
- `fallbackToDestructiveMigration` 사용 금지
- 외부 사용 분석 없음
- Android TextToSpeech 미국식 음성
- 사용자 지정 시각 하루 1회, 약 ±10분 알림
- 개인 API 키는 `local.properties`
- 공개 전 Gemini 호출을 백엔드로 이전

## 성과

- 연속 학습일
- 누적 학습 단어 수
- 상세 성과 지표는 이후 검토

## 협업

- Codex는 로직 담당
- Claude는 UI 담당
- 한 번에 한 에이전트만 실행
- 단일 `main` 브랜치
- 완료된 작업은 검증 후 커밋·푸시
- `CURRENT.md`, 요청 파일, 인수인계 문서로 상태 공유
