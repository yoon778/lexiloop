# 개인 맞춤 영단어 학습 앱 기획서

## 1. 제품 개요

말해보카와 비슷한 반복 학습 경험을 제공하되 영단어 암기에만 집중하는 Android 앱

문법 강의, 회화 강의, 랭킹, 친구, 과도한 게임 요소를 제거하고 짧고 반복 가능한 단어 학습에 집중

초기에는 개발자 본인이 1개월 동안 개인 베타로 사용하며 오류와 불편을 수정한 뒤 Play Store 출시 검토

## 2. 목표 사용자

- 먼저 개발자 본인
- 이후 개인 관심사에 맞는 영단어를 공부하려는 Android 사용자
- 일상 영어와 개발·업무 영어를 함께 학습하려는 사용자

## 3. 핵심 가치

- 사용자가 자연어로 학습 목적 설명
- 관심 분야와 난이도에 맞는 단어장 구성
- 아는 단어는 빠르게 검증
- 모르는 단어는 단계적으로 반복
- 학습은 오프라인에서 계속 가능
- 한 화면 한 행동의 깔끔한 경험

## 4. 온보딩

### 4.1 학습 목적 입력

예시

> 일상생활에도 사용하고 개발자 관점에서 자주 사용하는 단어 위주로 공부하고 싶어

AI가 다음을 추출

- 관심 분야
- 분야별 비율
- 예상 난이도
- 제외 분야
- 예시 단어

사용자가 생성 버튼을 누르면 최종 확인 팝업 표시

- 관심 분야와 비율
- 난이도
- 하루 신규 단어 수
- 예시 단어
- `취소`, `이대로 생성`

### 4.2 수준 진단

1. 초급, 중급, 고급 자가 선택
2. 진단 단어 20개 제시
3. `안다`, `헷갈린다`, `처음 본다` 선택
4. 결과를 초기 추천 난이도에 반영

### 4.3 하루 학습량

- 10개, 20개, 30개 또는 직접 입력
- 기본값 20개
- 신규 단어와 복습 단어 수를 분리

## 5. 개인 맞춤 단어장

### 5.1 생성 방식

혼합 구조 사용

- AI: 사용자 의도 분석, 단어 선정, 분야·난이도 분류, 부족한 예문 생성
- 검수된 사전 데이터: 뜻, 품사, 발음기호
- Android TextToSpeech: 미국식 발음

초기 추천 단어 300개 생성

대기열이 100개 미만이면 300개 추가 생성을 사용자에게 제안

### 5.2 여러 관심 분야

예: 일상 50%, 개발 50%, 하루 20개

- 일상 10개
- 개발 10개
- 출제 순서는 무작위 혼합
- 후보 부족 시 다른 분야에서 보충
- 최근 7일 기준 전체 비율 보정

### 5.3 관심사 변경

- 학습·복습 기록 유지
- 아직 학습하지 않은 추천 대기열만 갱신
- 학습한 단어는 기존 복습 일정 유지
- 제외한 단어는 다시 추천하지 않음

### 5.4 난이도 조절

- 앱이 자동으로 난이도를 바꾸지 않음
- 너무 쉽거나 어려운 경향이 있으면 조정 팝업 제안
- 사용자가 `쉽게`, `유지`, `어렵게` 선택
- 승인 후 추천 대기열만 변경

## 6. 학습 항목

학습 단위는 단순 철자 하나가 아니라 `표현 + 목표 뜻 + 문맥`

포함 대상

- 단어
- 숙어
- 구동사
- 개발 용어
- 여러 단어로 된 표현

### 6.1 여러 뜻

- 현재 학습할 목표 뜻 1개를 강조
- 다른 뜻은 흐린 색과 보조 표기로 표시
- 문제 정답은 목표 뜻 기준
- 다른 의미는 이후 별도 카드로 학습 가능

### 6.2 변화형과 파생어

- 기본형을 목표 항목으로 학습
- 변화형은 관련 표현으로 보조 표시
- 문장 문제에서는 문법상 필요한 정확한 형태만 인정
- 의미나 품사가 다른 파생어는 별도 카드

## 7. 신규 학습 흐름

### 7.1 전체 훑기

오늘의 신규 단어 목록을 빠르게 확인

### 7.2 개별 카드

한 장씩 표시

- 영어 표현
- 목표 뜻
- 보조 뜻
- 품사
- 발음기호
- 예문과 해석
- 미국식 음성 자동 재생과 다시 듣기

사용자 자기평가

- `안다`
- `헷갈린다`
- `처음 본다`

### 7.3 적응형 문제

일반 흐름

1. 영어 표현에서 한국어 뜻 선택
2. 한국어 뜻에서 영어 표현 선택
3. 한국어 뜻에서 영어 철자 입력
4. 문장 빈칸에 영어 표현 입력

`안다` 선택 시

- 객관식 생략
- 철자 입력과 문장 빈칸으로 실제 기억 검증
- 둘 다 정답이면 장기 복습 단계 이동
- 실패하면 일반 학습으로 복귀

오답 시

- 뜻과 예문 확인
- 현재 단계 또는 쉬운 단계로 복귀
- 세션 후반에 다시 출제

### 7.4 철자 판정

- 영어 철자 완전 일치 필요
- 대소문자 차이 무시
- 공백 차이 무시
- 한국어 뜻 타이핑 없음
- 2회 실패 시 첫 글자, 글자 수, 뜻 힌트 제공

### 7.5 문장 문제

- 짧고 자연스러운 문장
- 한국어 해석 표시
- 빈칸에 영어 표현 타이핑
- 힌트: 첫 글자, 글자 수, 뜻

### 7.6 객관식 오답 선택지

- 같은 품사
- 비슷한 난이도
- 철자나 의미가 적당히 유사
- 정답의 동의어 제외
- 최근 학습 항목 일부 활용
- 정답 위치 무작위

## 8. 문제 피드백

- 정답: 약 0.7초 표시 후 자동 진행
- 오답: 정답, 뜻, 예문 표시 후 사용자가 `다음` 선택
- 오답 화면에서 미국식 발음 자동 재생

## 9. 복습

### 9.1 간격

```text
학습 당일 -> 1일 -> 3일 -> 7일 -> 14일 -> 30일
```

- 전부 정답: 다음 간격
- 일부 오답: 이전 간격
- 완전 실패: 1일 단계
- 30일 통과: 장기 기억 단어
- 이후 60일 확인 가능

### 9.2 적응형 복습

- 철자 입력 또는 문장 빈칸부터 시작
- 정답이면 해당 복습 완료
- 오답이면 뜻 확인, 객관식, 철자 재시험
- 같은 날 다시 맞혀도 다음 복습 간격 단축

### 9.3 밀린 복습

- 예정 복습이 21개 이상이면 신규 학습 잠금
- 복습 처리 후 20개 이하가 되면 신규 학습 해제
- 오래 밀린 항목부터 출제
- 홈에서 잠금 이유와 남은 수 표시

## 10. 신규 학습량 처리

- 신규 목표 20개 중 이미 아는 단어가 있어도 다른 단어를 당일 추가하지 않음
- 확인한 20개 자체를 당일 신규 학습량으로 계산
- 다음 날 새 단어에서 자연스럽게 보충

## 11. 단어 관리

- `이미 완전히 앎`: 장기 복습으로 이동
- `나중에 다시`: 오늘 학습에서 제외
- `단어장 제외`: 다시 추천하지 않음
- `뜻·예문 오류`: 개인 베타에서는 로컬 메모 저장

## 12. 성과

MVP에서 표시할 항목만 유지

- 연속 학습일
- 누적 학습 단어 수

연속 학습일 인정 조건

- 당일 예정 복습 완료 또는 신규 목표 완료
- 단어 일부만 보고 종료하면 인정하지 않음
- 기기 현지 날짜 기준

## 13. 알림

- 사용자가 시각 지정
- 하루 1회 알림
- 설정 시각 기준 약 ±10분 허용
- 정확 알람 특수 권한 사용하지 않음
- 재부팅과 시간대 변경 시 재예약
- 알림 끄기 가능

## 14. 오프라인과 데이터

- AI 단어장 생성·재생성만 인터넷 필요
- 생성된 단어, 뜻, 예문, 진도는 기기 저장
- 학습과 복습은 오프라인 지원
- AI 실패 시 내장 기본 일상 단어장 제공
- 자동 무한 재시도 없음
- 기존 단어장은 실패 시 유지

개인 베타

- 로그인 없음
- 클라우드 동기화 없음
- JSON 내보내기·가져오기 지원
- 외부 행동 분석 없음
- 오류 로그는 기기 내부 저장과 내보내기만 지원

## 15. 데이터 초기화

각 기능 분리 후 확인 팝업 표시

- 학습 기록만 초기화
- 추천 단어장만 재생성
- 제외 단어 복원
- 전체 데이터 삭제

## 16. AI 운영

개인 베타

- 본인 Gemini API 프로젝트 사용
- 개인 키는 `local.properties`에 저장
- 학습 중 AI 호출 없음
- 50개 단위로 생성하고 검증 후 저장

공개 버전

- 앱 직접 API 호출 제거
- 백엔드에서 Gemini 호출
- 사용자별 생성 횟수 제한
- Google 로그인은 계정·동기화 용도이며 Gemini 비용과 분리

## 17. 영한 데이터 원칙

- 오픈 라이선스 영한 데이터 우선
- AI로 목표 뜻과 보조 뜻 초안 보완
- 개인 베타 중 직접 수정 가능
- 출처와 라이선스 기록
- Play Store 출시 전 상업적 재사용 가능 여부 검토
- 사전 웹사이트 무단 크롤링 금지

## 20. 개인 베타 목적

1개월은 학습 진도 달성 시험이 아니라 제품 QA 기간

- 불편한 흐름 수정
- 데이터·예문 오류 수정
- 필요한 기능 추가
- 필요 없는 기능 제거
- 안정화 후 Play Store 출시 검토
